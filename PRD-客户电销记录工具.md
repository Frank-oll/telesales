## 一、产品背景与目标

### 1.1 真实使用场景还原
> 我从公司拿到一份客户名单，坐下来开始逐个打电话。一通电话 30 秒到几分钟不等，挂掉后我要立刻拨下一个。我希望工具能做到：
> - **每通电话挂断后 3 秒内**就能记完，不耽误下一通
> - 边打边看到 **本批次的实时统计**（已打多少通、其中多少意向）
> - 名单不固定长度，打完了我点"结束"就行
> - 结束后系统自动给我一份**可展示给老板的汇总图**，以及一份**意向客户名单 Excel**

### 1.2 产品目标
1. **极致快速录入**：4 个标签按钮，一键完成大多数非意向通话的记录
2. **实时反馈**：侧边栏统计随每次提交即时更新
3. **批次自治**：会话起止由用户控制，每次结束自动产出物
4. **轻交付**：汇总图（社交/汇报场景）+ 意向 Excel（业务跟进场景）

### 1.3 非目标（v1 明确不做）
- ❌ 客户档案中心 / 跨批次客户跟进
- ❌ 历史会话查询与对比
- ❌ 多人协作 / 云同步
- ❌ 回访提醒 / 自动外呼
- ❌ 通话录音管理
- ❌ 多维漏斗（v1 只做四类标签 + 意向二分）

## 二、核心使用流程

### 2.1 完整流程图

```
[打开应用]
    ↓
[自动进入"本次电销"工作台]
    ↓
   ┌──────────────────────────────────────────────┐
   │  循环：每通电话挂断后                          │
   │                                              │
   │   选择本次通话结果标签                         │
   │   ├─ [直接挂]  → 记录1条统计，刷新右侧 → 下一通│
   │   ├─ [未接]    → 记录1条统计，刷新右侧 → 下一通│
   │   ├─ [已有供应商] → 弹出客户信息表单 → 提交     │
   │   └─ [待报价]  → 弹出客户信息表单 → 提交       │
   │                                              │
   │  （右侧统计面板实时更新）                      │
   └──────────────────────────────────────────────┘
    ↓
[点击「结束本次电销」按钮]
    ↓
[自动跳转 → 本次会话汇总页]
    ├─ 总体数据胶囊（总拨打 / 接通 / 意向数）
    └─ 意向客户列表
    ↓
    ├─ [导出汇总图片]  → PNG
    └─ [导出意向 Excel] → XLSX（仅在有意向客户时可点击）
    ↓
[关闭汇总页 → 回到空白工作台，准备下一批次]
```

### 2.2 关键状态机
- **会话生命周期**：打开应用 → 录入中 → 结束 → 汇总展示 → 关闭汇总
- 一次应用打开周期内可进行多个会话（结束后回到空白工作台即开始新会话）

## ★ 本次迭代变更（v2.1 · 对应 App v1.2.0）

> 本章节为本轮新增需求的**权威说明**。下文第三章「界面与模块设计」中与之冲突的旧描述（如旧的两栏布局、"已打 N 个电话"角标）以本章节为准。
> 改动只发生在**主工作台**与**历史会话页**两处，其余流程（意向登记表单、汇总页、跟进页、导出格式、数据存储）保持现状不变。

### A. 删除"已打 N 个电话"角标

- **现状**：主工作台"本通电话结果"标题右侧有一行"已打 N 个电话"实时计数（DOM 中 `#totalCountInline`，由 `renderWorkspace()` 刷新）。
- **改动**：整体删除该角标。
- **理由**：右侧黑卡"本次累计拨打"已展示同一数字，角标重复且占用标题行空间；删除后标题行更干净，也为下方按钮区让出视觉重心。
- **影响**：删除对应 DOM 与 `setText('totalCountInline', …)` 调用即可，统计数据源（`state.calls.length`）不受影响。

### B. 四个标签按钮：高度收敛 + 整体居中

- **现状**：四个按钮各自 `flex-1` + `min-h-96px`，纵向撑满整张卡片（卡片高度随右侧统计列拉伸），按钮间被拉得很高。
- **改动**：
  - 按钮高度收敛（建议每个 `min-h` 56–64px 区间，最终以实现微调为准），不再强行撑满卡高；
  - 按钮组在卡片内**整体垂直居中**，上下留白；文字保持水平 + 垂直居中；
  - 四按钮之间的分隔线保留；
  - 卡片本身随该行高度拉伸，与右侧统计列**等高**（左侧「近期标记」为紧凑面板、左上对齐，不参与等高拉伸，详见 C 节）。
- **理由**：按钮过高时点击落点漂移、视觉松散；收敛后更紧凑、命中更稳，也与新增的左侧模块形成均衡的三列版式。
- **标签文案保持现状**：暂无磁铁需求 / 未接 / 已有供应商 / 待报价（键盘快捷键 1/2/3/4 不变）。

### C. 工作台左侧新增「近期标记」模块

主工作台由原「**左录入(2) + 右统计(1)**」两栏，改为「**左·近期标记 + 中·本通电话结果 + 右·实时统计**」三栏布局（`md:grid-cols-4`，比例 1 : 2 : 1）。中栏与右栏等高拉伸；**左栏「近期标记」为紧凑面板、左上对齐，固定限高 + 内部滚动，不随记录数把其余两栏撑高**。

```
┌────────────────────────────────────────────────────────────────────────┐
│  客户电销记录                          [跟进] [历史] [结束本次电销]        │
├──────────────────┬──────────────────────────┬──────────────────────────┤
│  ① 近期标记       │   ② 本通电话结果          │   ③ 实时统计(现状不变)    │
│  (新增·可滚动)    │   (按钮收敛+居中)         │                          │
│                  │                          │   ┌────────────────────┐ │
│  ● 待报价 张三    │       暂无磁铁需求         │   │ 本次累计拨打   58   │ │
│    31 秒前       │  ───────────────────     │   └────────────────────┘ │
│  ● 未接          │         未接              │   本次明细 / 营销质量      │
│    1 分钟前      │  ───────────────────     │   意向客户 / 转化率        │
│  ● 暂无磁铁需求   │       已有供应商          │                          │
│    2 分钟前      │  ───────────────────     │                          │
│  ● 已有供应商王五 │        待报价            │                          │
│    …(最多4条)    │                          │                          │
│  ↕ 滚轮看更旧     │                          │                          │
└──────────────────┴──────────────────────────┴──────────────────────────┘
```

**模块定义**

| 项 | 说明 |
|---|---|
| 标题 | "近期标记"（标题行可附"共 N 次"小字计数，可选） |
| 数据源 | `state.calls`，**倒序**渲染（最新在最上，依次向下变旧） |
| 单条内容 | 类型彩点 + 类型名 + 相对时间（如"31 秒前"）；意向类（已有供应商 / 待报价）额外显示**客户姓名**（按 `at` 关联 `state.customers`） |
| 彩点配色 | 复用右侧明细色：暂无磁铁需求 `#8e8e93`、未接 `#ff9f0a`、已有供应商 `#0a84ff`、待报价 `#30d158` |
| 模块形态 | **紧凑面板**，左上对齐（`self-start`，不参与三列等高拉伸）；高度 = 表头 + 列表限高，避免大量记录把中/右两栏撑高 |
| 行高 | **固定舒适行高**（约 50px），行间细分隔线 |
| 可见容量 | 默认**最多同时可见最近 4 条**（列表 `max-height ≈ 4 行`） |
| 溢出处理 | 超过 4 条的更旧记录**隐藏在下方**；鼠标移入模块后，向下滚轮即可查阅更旧记录（内部 `overflow-y:auto`） |
| 实时刷新 | 每次点击标签 / 提交意向客户后，新记录**从顶部插入**，并可给一次轻微高亮（沿用 `pulse`，可选） |
| 空态 | 尚无标记时显示占位："开始拨打后，这里会实时记录每一次标记" |
| 交互 | 本期**只读展示**，不做点击跳转 / 编辑（如需后续再议） |

**响应式**：桌面（md+）三列横排，左→右 = 近期标记 / 本通电话结果 / 实时统计；窄屏（单列）按 DOM 顺序纵向堆叠（移动端为次要场景，本工具主用桌面 Electron 窗口）。

### D. 历史会话：多选合并 + 合并导出

- **现状**：历史页（`#history`）以会话卡列出归档会话，点单张卡进入只读汇总页（`viewHistorySession` → `state.viewing` → 汇总页）；汇总页支持导出图片 / 意向 Excel（作用于单个会话）。
- **改动**：新增"多选 → 合并 → 合并导出"能力。

**交互流程**

1. 历史页右上新增"**多选**"开关（或每张会话卡前置复选框）；进入多选态后会话卡可勾选。
2. 选中 **≥2** 条会话时，出现"**合并查看（N）**"操作按钮。
3. 点击后生成**合并数据集**（所选会话的 `calls`、`customers` 合并），进入**复用的只读汇总视图**：
   - 标题："合并电销汇总报告"；
   - 副标题："合并 N 次会话 · 最早日期 ~ 最晚日期"；
   - 3 个数据胶囊：合并后的总拨打 / 接通 / 意向；
   - 意向客户列表：合并后的全部意向客户（只读，行不可编辑）。
4. 合并视图支持现有两个导出按钮，作用于合并数据集：
   - **导出汇总图片（PNG）**：文件名 `电销合并汇总_最早日期_最晚日期.png`；
   - **导出意向客户 Excel（XLSX）**：在现有列基础上**新增"来源会话日期"列**，便于区分跨会话客户；文件名 `电销合并意向客户_最早日期_最晚日期.xlsx`。
5. 合并视图为**临时只读视图**：不落库、不改动原会话归档；返回即丢弃。

**默认决策（需你确认，可调整）**

| # | 议题 | 默认方案 |
|---|---|---|
| D-1 | 合并时是否按手机号去重 | **不去重**，全部列出；以"登记时间 + 来源会话日期"区分重复号码 |
| D-2 | 合并入口形态 | 历史页右上"多选"开关 + 会话卡复选框 + 选中 ≥2 后出现"合并查看(N)"按钮 |
| D-3 | 合并范围上限 | 不设硬上限（可多选任意数量会话） |

**交互细节**
- 4 个标签按钮采用大尺寸、高对比色块，可键盘快捷键操作（1/2/3/4）
- 非意向标签（直挂/未接）点击后**无弹窗、无确认**，直接计入统计并给一个 0.5 秒视觉反馈（按钮闪一下、统计区数字跳动）
- 意向标签（已有供应商/待报价）点击后弹出客户信息登记表单
- 右侧统计面板与左侧录入区**同屏可见**，所有数据实时联动
- 「结束本次电销」按钮置于右上角，醒目但需二次确认

### 3.2 意向客户登记表单（弹窗）

仅在选择「已有供应商」或「待报价」时弹出。

**字段定义**

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 意向类型 | 自动带入 | 是 | 已有供应商 / 待报价（来自上一步标签） |
| 客户姓名 | 文本 | 是 | |
| 联系电话 | 文本 | 是 | **软校验**：非 11 位手机号格式给提示但不阻断；本次会话内做电话去重提示（重复时弹"该号码本次已登记过"） |
| 公司名称 | 文本 | 否 | |
| 职位 | 文本 | 否 | |
| 所在地区 | 文本 | 否 | |
| 谈话要点 | 长文本 | 否 | 客户关注点、**报价金额/区间**、现有供应商情况等（不单设"报价"字段，统一在此自由记录） |
| 下次跟进点 | 短文本 | 否 | 自由填写 |
| 备注 | 长文本 | 否 | |

**交互细节**
- 弹窗居中、宽度适中，键盘 Enter 跳下一字段、Ctrl+Enter 提交
- 提供「取消」按钮 —— 取消即不计入任何统计（避免误点）
- 提交后弹窗关闭，回到主工作台，右侧统计即时刷新

### 3.3 实时统计面板（右侧固定栏）

**展示指标**

| 指标 | 计算口径 | 实时刷新时机 |
|---|---|---|
| 总拨打数 | 4 类标签提交数总和 | 每次提交 |
| 直接挂 | 标签提交计数 | 每次提交 |
| 未接 | 标签提交计数 | 每次提交 |
| 已有供应商 | 标签提交计数（含客户表单） | 客户表单提交后 |
| 待报价 | 标签提交计数（含客户表单） | 客户表单提交后 |
| 接通数（隐式） | 总拨打 - 未接 | 实时 |
| 意向客户数 | 已有供应商 + 待报价 | 实时 |
| 意向转化率 | 意向数 / 总拨打数 | 实时 |
| 本次开始时间 | 进入工作台的时间 | 一次性 |
| 已耗时 | 当前时间 - 开始时间 | 每分钟刷新 |

### 3.4 「结束本次电销」按钮

- 位置：主工作台右上角，醒目色（如蓝色或绿色）
- 点击后弹窗二次确认："确认结束本次电销？结束后将生成本次汇总报告。"
- 确认后立即跳转汇总页，主工作台数据冻结但不清空（汇总页关闭后才清空）

**边界处理**
- 若本次会话**一通都没打**就点结束 → 提示"本次未记录任何通话，无需生成报告"，并询问是否真的结束（结束则直接回到空白工作台）
- 若意外关闭应用 → **v1.0 不做断点恢复**，下次打开即新会话（断点恢复列入 v1.1 范围）

## 四、数据存储

### 4.1 存储策略
- v1.0 默认**保留所有历史会话数据到本地 SQLite**（防误关、未来回查），但**界面不提供历史查询入口**，也**不提供清理入口**
- 数据文件位置（macOS）：`~/Library/Application Support/客户电销记录/data.db`
- 若需手动清空，用户自行删除该 db 文件即可

### 4.2 数据模型（简化版）

```
sessions（会话表）
  - id
  - started_at
  - ended_at
  - status: in_progress / ended
  - total_calls / hangup_count / unanswered_count
    / has_supplier_count / pending_quote_count

intent_customers（意向客户表）
  - id
  - session_id (FK)
  - intent_type: has_supplier / pending_quote
  - name / phone / company / position / region
  - talking_points / next_followup / remarks
  - created_at
```

> 注：非意向标签（直挂、未接）只增加 sessions 表中的计数，不单独存为客户记录 —— 保持数据精简。

## 五、技术方案

### 5.1 技术栈

| 层 | 选型 | 理由 |
|---|---|---|
| 应用框架 | Electron | 跨平台桌面、Web 技术栈 |
| 前端框架 | React + TypeScript | 组件化、类型安全 |
| UI 组件库 | shadcn/ui 或 Ant Design | 表单、弹窗、按钮开箱即用 |
| 本地存储 | better-sqlite3 | 单文件 SQLite，零依赖、性能好 |
| 图片导出 | html2canvas | DOM 转 PNG 主流方案 |
| Excel 导出 | SheetJS（xlsx） | 主流方案，无需 Excel 安装 |
| 打包 | electron-builder | 出 dmg 安装包 |

### 5.2 关键技术点

| 难点 | 解决方案 |
|---|---|
| 实时统计性能 | 用 Zustand 等轻量状态管理，提交即更新内存状态 + 异步落库 |
| 图片导出清晰度 | html2canvas 设置 scale=2，确保高分辨率 |
| 中文文件名 | 导出时统一处理编码，避免乱码 |
| 防误关 | 工作台内任意操作发生时，监听 `beforeunload`，关闭时提示"本次会话尚未结束，是否确认关闭？" |
